# 正EV交易决策台

一个用于**下单前**的支撑阻力、进出场与风险检查工具。它不负责预测涨跌，而是把多个交易框架对“位置、触发、失效和目标”的判断放在同一张地图里。

## 当前能力

- 多周期行情：1m、5m、15m、1h、4h、1d
- 市场：主流加密资产、美股、US Tech 100（Nasdaq-100）、黄金与原油；非加密市场默认读取每 30 分钟更新的闭合 K 线快照，可优先接 Alpaca
- 统一行情服务：闭合 K 线、24H 价格、资金费率、OI、24H OI 变化与宏观上下文
- 下单前位置地图：将多种价位来源聚合为支撑/阻力区
- 条件式情景：支撑确认、阻力确认、突破回踩、跌破反抽
- EV 门控：方向、位置、赔率同时成立才允许记录计划；在已验证范围再经过低频训练门控否决
- 风险仓位：按账户规模与单笔风险计算名义仓位
- 多流派评审团：可选择专家并调整权重
- 历史战绩：17 位方法教练逐一展示当前范围战绩、跨市场训练结果或未参赛原因
- 今日机会榜：缓存自选池扫描结果，按位置、赔率、共识、可信度、行情适配和冲突从高到低排序
- 本地交易日志：用 R 值记录结果并计算纪律分
- Paul Wei 行为证据：公开教师账本、动作结构和已核验的行为结果

## 18 个专家身份

系统将 17 个可复现的方法视角与 1 个外部行为模型严格分开。Paul Wei 不被伪装成第 18 个规则指标，而是通过版本化专家观点接口进入当前观点和前瞻赛场。

| 类别 | 方法 | 主要回答 |
|---|---|---|
| 趋势结构 | 趋势跟踪、道氏理论、一目均衡 | 当前更适合顺势、逆势还是等待 |
| 价格行为 | Al Brooks 价格行为 | 趋势K、交易区间与触发K如何组合 |
| 量价行为 | 威科夫、成交量分布、锚定 VWAP | 市场接受价、成本区与区间边缘反应 |
| 结构与流动性 | SMC、ICT 流动性 | BOS、扫损、前高前低与 FVG |
| 东方结构 | 缠论 | 中枢、背驰与潜在买卖点 |
| 价位工具 | 支撑阻力聚类、斐波那契 | 哪些价位是高汇合决策区 |
| 统计指标 | 均值回归、动量因子 | 行情是否偏离均值或存在持续动量 |
| 执行与环境 | 网格、资金费率情绪、宏观 | 当前策略是否适配、是否存在拥挤风险 |
| 行为模型 | Paul Wei 重建 | 先判断是否应行动，再区分进攻与降风险；具体价位由 EV Desk 独立生成 |

不同方法并非完全独立。工具会展示汇合来源，但不会把高度相关的多条线解释成多份独立证据。

## 价位地图

价位候选来自：

- 多周期摆动点聚类
- 唐奇安区间与 60 根摆动高低
- 前周期高、低与经典枢轴
- EMA20 / EMA50
- Bollinger 上、中、下轨
- Fibonacci 0.382 / 0.5 / 0.618 / 0.786
- 成交量 POC / HVN（由 K 线典型价近似）
- 重要波段锚定 VWAP
- 整数心理关口

相近价位按 ATR 容差聚合为“区”，再按照来源权重、重复测试与来源多样性评分。分数表示**汇合强度**，不是胜率。

## 决策顺序

1. 先看多周期行情环境。
2. 再看现价离最近强支撑/阻力多远。
3. 选择主情景与备选情景。
4. 等待收回、拒绝、突破回踩等明确触发。
5. 把止损放在结构失效区外。
6. 目标取下一处有意义的汇合区并计算 R:R。
7. 方向、位置或赔率任一不成立，不下单。

## 专家前瞻赛场

历史回测与前瞻模拟严格分开。赛场只从用户启用之后开始记录：

1. 每根已收盘K线最多为每位专家封存一个新计划。
2. 封存字段包括专家、标的、周期、方向、行情环境、进场、止损、目标、触发与失效条件。
3. 挂单最多等待12根K线；成交后最多持有30根K线。
4. 同根K线同时触及止损和目标时，按悲观规则先止损。
5. 每笔扣除0.10%的往返成本，并换算为R。
6. 已结算结果进入不可由界面单独删除的审计账本，可完整导出JSON。
7. 专家状态按前瞻样本显示为“热身中、观察期、前瞻有效、基本稳定或降权观察”。

动态权重只使用前瞻结果，并对小样本进行强收缩：少于5笔不调整；此后逐渐依据EV和回撤在0.65x–1.35x范围内校准。所谓“成长”指近期EV相对此前样本的变化，以及对适用行情环境的识别，**不会为了美化结果自动重写方法规则**。

## 三层证据

- 当前观点：最新闭合 K 线下的方向、理由、计划、版本和数据时间。
- 历史证据：固定规则的历史模拟，展示 95% 区间、EV、Profit Factor、最大回撤和环境漂移。
- 前瞻成长：观点先封存再结算，只有这一层用于动态权重和成长判断。

首批正式历史证据覆盖 BTC/ETH/SOL 的 15m、1h、4h，并由 `data/expert-evidence.json` 与 `data/expert-evidence.js` 提供。Paul Wei 的最新行为评分、样本外方向校准和外部观点批次位于 `data/`。

## 长历史训练证据

用户提供的本地档案已形成第二套、更长的教练训练证据：

- Paul Wei：173,058 条执行记录、43,214 张订单、98,777 条 XBTUSD 交易执行，形成 4,825 个行为片段，其中 4,822 个可由本次 XBTUSD 账本核验。
- BTC/ETH 永续：各 2,890,080 根完整 1 分钟 K 线，覆盖 2021-01-01 至 2026-06-30。
- NQ：1,048,575 根 1 分钟数据，覆盖 2022-12-26 至 2025-12-11；文件恰好达到 Excel 数据行上限，因此明确标记可能截断。
- ES：从混合 ES/MES/价差/远月文件中筛出 355,021 根主力连续 ES 分钟数据，并记录 4 次移仓调整。
- 共准备 12 个 `标的 × 周期` 范围，覆盖 BTC、ETH、NQ、ES 的 15m、1h、4h。
- 14 个可独立产生方向的方法镜头完成了最终压力测试。网格是执行方式；情绪缺完整历史资金费率；宏观缺利率、美元与流动性序列，因此这 3 位明确标记为未参赛，不用 0 战绩填补。
- 简单多数投票在开发、验证和最终留出均为负，因此未部署。
- 从 248,295 个已执行候选计划训练的浅层计划门控，仅在 `BTCUSDT|1h` 与 `ETHUSDT|4h` 同时通过验证和最终留出要求。最终留出 251 笔，范围加权 EV 为 `+0.064R`，但只作为受限的安全否决层，绝不生成方向、价位或仓位。

可发布聚合位于：

- `data/coach-training.json` / `data/coach-training.js`
- `data/plan-gate-model.json` / `data/plan-gate-model.js`

原始订单、执行 ID 与分钟数据由 `.gitignore` 排除，不会上传到 GitHub。完整方法、质量问题和数据血缘见 `docs/DATA_AND_EXPERT_EVIDENCE.md`。

## 数据与限制

- 页面偏好与个人日志保存在 localStorage；公共行情、专家快照和前瞻账本由 `arena-worker/` 提供。
- 本地开发会自动连接 `http://localhost:8790`；生产版在 `config.js` 配置 Worker origin。
- Worker 未运行或连接失败时，前端自动使用浏览器行情；已发布的历史战绩和训练结果仍可查看。
- 宏观数据需要 FRED key；美股生产行情可配置 Alpaca。未配置时，美股、US Tech 100、黄金与原油读取 GitHub 定时发布的 Yahoo Finance 延迟快照，并保留浏览器直连备援。
- 浏览器端公共 API 可能受地区、CORS 或限流影响。
- K 线成交量分布是近似值，不等同交易所逐笔成交或完整订单簿。
- Brooks、ICT、威科夫和缠论等依赖上下文的框架被规则化简，仅适合作为结构化检查项。
- 仅对已有历史数据的“专家 × 标的 × 周期”显示回测；没有数据时明确标记待接入。
- 训练门控仅授权 BTC 1h 与 ETH 4h；其他标的/周期仍可查看规则计划，但页面明确标记其没有训练授权。
- 历史回测和汇合分数均不保证未来表现。本项目是分析与教育工具，不是投资建议。

## 运行

页面本身无构建依赖：

```bash
python3 -m http.server 8780
```

然后访问 `http://localhost:8780/app.html`。要启用统一实时数据和 24H 赛场，同时在 `arena-worker/` 运行：

```bash
npm ci
npm run db:local
npm run dev -- --port 8790
```

完整数据矩阵、Paul Wei 更新方式、Cloudflare 部署、密钥申请和专家接口见 `docs/DATA_AND_EXPERT_EVIDENCE.md`；Worker API 见 `arena-worker/README.md`。

本地原始档案齐备时，可一键重建教练训练证据与计划门控：

```bash
tools/train_coaches.sh "/path/to/enent contract"
```

脚本按时间顺序划分开发、验证与最终留出，不随机打乱；模型阈值不读取最终留出结果。
